# Product Owner

## Odpowiedzialność
- Wizja produktu ADHD-friendly.
- Backlog i priorytety funkcjonalności.
- User research z perspektywy osób neuroatypowych.

## Kluczowe decyzje
1. **Emergency Mode** - nieusuwalne streaki i celebracja minimum.
2. **ADHD-First UX** - brak kar, więcej dopaminy z małych sukcesów.
3. **Frictionless Onboarding** - minimalna konfiguracja przed pierwszą wartością.

## Metryki sukcesu
- Procent użytkowników wracających po 7 dniach.
- Liczba ukończeń na poziomie Emergency.
- Brak skarg dotyczących zresetowanych streaków.
